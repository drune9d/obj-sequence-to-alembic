# OBJ Sequence to Alembic

Convert a folder of OBJ frames into an Alembic `.abc` mesh cache on macOS.

This fork adds a Homebrew/CMake macOS build, a tkinter GUI, a double-click
launcher, and a release packaging script that creates a downloadable `.app`
bundle for GitHub Releases.

## What You Get

| File | Purpose |
| --- | --- |
| `OBJ Sequence to Alembic.app` | App bundle generated for releases |
| `gui.py` | Tkinter interface for artists and non-terminal use |
| `bin/Objs2Abc` | C++ command line converter, generated by `build.sh` |
| `OBJ_to_ABC.command` | Double-click launcher for working from the repo |
| `build.sh` | Installs dependencies and builds the converter |
| `scripts/package_release.sh` | Builds a GitHub Release zip |

## Download And Use

For normal users, download the latest `OBJ-Sequence-to-Alembic-macOS.zip` from
GitHub Releases.

1. Unzip the download.
2. Open `OBJ Sequence to Alembic.app`.
3. Choose the folder containing your OBJ sequence.
4. Choose an output folder, or leave it blank to create `output` next to the
   input folder.
5. Choose the `.abc` filename, framerate, and node name.
6. Click `Convert`.

The app shows the detected frame count, first and last OBJ file, first-frame
vertex/face counts, and whether UVs/normals were found in the first frame.

## Requirements

- macOS 13 or newer
- Homebrew
- Python 3 with tkinter

The build script installs the required Homebrew packages when missing:

```bash
brew install cmake alembic hdf5 imath zlib
```

The release app includes the converter binary, but it is still linked against
Homebrew Alembic/HDF5/Imath/zlib. If conversion fails because libraries are
missing, install Homebrew and click `Rebuild` in the app.

## Use From This Repo

From a cloned checkout, the easiest path is:

```bash
./build.sh
open .
```

Then double-click:

```text
OBJ_to_ABC.command
```

The `.command` launcher builds the converter if needed, then opens the GUI.

## Command Line

```bash
bin/Objs2Abc -i <input_obj_folder> -o <output_file.abc> -f <framerate>
```

Example using the sample data:

```bash
bin/Objs2Abc -i head-poses -o output/head-poses.abc -f 24 -n Head
```

## Make A GitHub Release Zip

For maintainers:

```bash
scripts/package_release.sh
```

This creates:

```text
dist/OBJ-Sequence-to-Alembic-macOS.zip
```

Upload that zip to a GitHub Release. The zip contains:

- `OBJ Sequence to Alembic.app`
- `README.md`
- `LICENSE`

The app bundle contains the GUI, converter binary, source needed for rebuilds,
and the `head-poses` sample sequence.

## OBJ Sequence Notes

Best results come from clean mesh-cache style OBJ sequences:

- One folder per sequence.
- Sortable frame names such as `0000.obj`, `0001.obj`, `0002.obj`.
- Same topology across all frames.
- Same UV layout across all frames.
- One mesh per OBJ file.

The first OBJ frame defines the Alembic topology and UV layout. Later frames are
expected to update the same mesh's vertex positions.

## Data Support

The converter writes an Alembic `OPolyMesh` cache.

Preserved:

- Animated vertex positions
- Face indices and face counts
- Face-varying UVs when OBJ `vt` data exists
- Framerate/time sampling

Not currently preserved:

- Normals
- Materials or `.mtl` files
- Texture paths
- OBJ groups or object names
- Vertex colors
- Multiple independent meshes in one OBJ
- Custom attributes

In short: UVs are carried over when present, but normals and material data are
not written by the current converter.

## Troubleshooting

If Homebrew is missing, install it from:

```text
https://brew.sh
```

If the app says the converter is missing, click `Build Converter` or run:

```bash
./build.sh
```

If macOS blocks the app because it was downloaded from the internet, right-click
the app and choose `Open`.

If CMake cannot find a dependency, install the command it prints and run the
build again. The build supports Apple Silicon Homebrew at `/opt/homebrew` and
Intel Homebrew at `/usr/local`.
